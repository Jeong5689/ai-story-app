원인:
Firebase 보안 규칙에서 읽기/쓰기 허용 조건을 `request.auth != null`로 설정해 두었기 때문에, 인증되지 않은 클라이언트가 Firestore 접근을 시도하면 "Missing or insufficient permissions" 오류가 발생했습니다.

수정 내용:
1) `lib/authService.ts`
   - `signInAnonymously`를 import하고 `ensureAuth()` 함수를 추가함.
     - 역할: `auth.currentUser`가 없으면 익명으로 로그인하여 클라이언트가 인증 상태를 갖도록 보장합니다.

2) `lib/storyService.ts`
   - Firestore를 읽거나 쓰는 함수들(`saveStory`, `getUserStories`, `deleteStory`, `updateStory`)의 시작 부분에서 `await ensureAuth()`를 호출하도록 변경함.
     - 역할: Firestore 규칙(`request.auth != null`)을 만족시키기 위해 호출 시점에 인증을 보장합니다.
3) `app/generate/page.tsx`
   - `saveStory()` 호출 전에 `ensureAuth()`를 사용하여 현재 Firebase 인증 사용자 UID를 확보하도록 수정함.
     - 역할: React `currentUser`가 아직 로드되지 않았거나 익명 인증이 필요한 경우에도 Firestore 요청이 올바른 인증 정보로 이루어지도록 합니다.

이유 및 영향:
- 이유: 기존 코드에서는 UI에서 로그인 확인을 하지만(예: `currentUser` 검사), 일부 경로(또는 초기 상태)에서 인증이 누락된 채 Firestore 요청이 발생할 수 있어 권한 오류가 발생할 수 있습니다. 익명 인증을 보조로 사용하면, 인증이 필요한 규칙을 만족시키면서도 사용자 흐름을 방해하지 않습니다.
- 영향: 사용자가 명시적 로그인(이메일/비밀번호)을 하지 않아도 클라이언트가 익명 계정으로 인증되어 Firestore 접근이 가능해집니다. 기존 인증(로그인된 사용자)이 있으면 그 계정이 우선됩니다.

검증 방법:
1) 로컬에서 앱 실행 후 동화 생성/저장 기능을 시도합니다.
   - 기존에는 "Missing or insufficient permissions" 오류가 떴던 부분이 더 이상 발생하면 성공입니다.
2) Firebase 콘솔에서 Authentication > 익명(Anonymous) 항목에서 신규 익명 계정이 생성되었는지 확인합니다.
3) Firestore 규칙은 그대로 두어도 동작해야 합니다(`request.auth != null`).
4) 기존 이메일 로그인 사용자 흐름은 그대로 유지됩니다.

추가 수동 확인(문제가 계속될 경우):
- 브라우저 개발자 도구 Console 탭에서 `생성 오류 상세:` 또는 `[ensureAuth]` 로그를 확인하세요. 익명 로그인 실패 시 `auth/operation-not-allowed` 또는 유사한 에러 코드가 보일 수 있습니다.
- Firebase 콘솔 > Authentication > Sign-in method 에서 "Anonymous"가 활성화되어 있는지 확인하세요. 비활성화 상태면 익명 로그인은 실패합니다.
- Firestore 규칙이 `request.auth != null` 이외에 추가 조건(예: `resource.data.userId == request.auth.uid`)을 포함하고 있지 않은지 확인하세요. 추가 조건이 있으면 규칙에 맞게 클라이언트 요청(필드값)을 조정해야 합니다.

문제가 계속되면 알려주세요. 브라우저 콘솔의 오류 메시지(전체)를 붙여주시면 더 정확하게 원인 진단 후 추가 수정하겠습니다.
